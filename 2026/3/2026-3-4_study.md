# macro とは？

## 結論（Point）

`macro` は、**ActiveRecordの関連が「どの種類か（belongs_to / has_many / has_one など）」を表す情報**です。  
`reflect_on_association` と一緒に使うと、**モデルの関連定義が正しいかをテストで確認**できます。

---

## 理由（Reason）

Railsの関連は、設計の中心です。  
関連の種類が違うと、アプリの動きも変わります。

例：

- `has_one` と `has_many` を間違えると「1つだけ」のはずが「複数」にできてしまう
    
- `belongs_to` を消すと「必須のつながり」がなくなる
    

`macro` を見ると、**関連が意図した種類で定義されているか**を、DBを触らずにチェックできます。

---

## 具体例（Example）

Roomモデルの `issuer_profile` が `belongs_to` かどうかを確認します。

```rb
assoc = Room.reflect_on_association(:issuer_profile)
assoc.macro
# => :belongs_to
```

RSpecではこう書きます。

```rb
it "issuer_profile is belongs_to" do
  assoc = Room.reflect_on_association(:issuer_profile)
  expect(assoc.macro).to eq(:belongs_to)
end
```

これで、関連を `has_one` に変えてしまったり、削除したときにすぐ気づけます。

---

## 結論（Point 再掲）

`macro` は、**関連の種類を示すラベル**です。  
`reflect_on_association` と組み合わせると、**関連の設計ミスをテストで防げます**。
