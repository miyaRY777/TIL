# 今日理解できたこと
## RSpec

### リクエストスペックの責務

* リクエストスペックは、**「このURLにアクセスしたとき、正しいレスポンスになるか」**を確認するためのテスト

### `describe` と `context` の違い

* `describe` は、**何をテストするか**を書く
* `context` は、**どんな状況・条件でテストするか**を書く

### `response.body` とは

* `response.body` は、**サーバーが返したHTTPレスポンスの本文（中身）**を取り出している

### `reload` が必要な理由

```ruby
expect(user.reload.nickname).to eq("新ニックネーム")
```

* 手元の `user` オブジェクトと、データベースの中身がズレることがある
* そのため `reload` は、**レコードをデータベースから再読み込みするために必要**

## Hotwire

### `turbo frame` とは

* ページ全体ではなく、**指定した一部分だけを更新するための仕組み**
* `<turbo-frame>` で囲んだ範囲の中では、リンクのクリックやフォーム送信がその枠の中だけで処理される
* そのため、他の部分はそのまま保たれる

## Ruby on Rails

### `respond_to do |format|` とは

* 同じコントローラーアクションの中で、**リクエストされた形式ごとに返す内容を切り替える仕組み**

```ruby
format.html {}
```

* 普通のWebページを返すための形式
* 画面全体を表示し直したり、別ページへ遷移したりするときに使う

```ruby
format.turbo_stream {}
```

* Turbo Stream形式のレスポンスを返すための形式
* ページ全体ではなく、画面の一部だけを差し替えたり、追加・削除・更新したいときに使う

```ruby
format.json {}
```

* JSON形式のデータを返すための形式
* 画面そのものではなく、JavaScriptや外部クライアントに渡すデータを返したいときに使う

### `status: :unprocessable_entity` の意味

* `status: :unprocessable_entity` は、**HTTPステータスコード422を返す**という意味
* 422は、**フォーム送信自体は届いたが、入力内容に問題があって更新できなかった**ことを表す

### `as:` オプションが `GET` だけに付いている理由

```ruby
get "mypage/settings", ..., as: :mypage_settings
patch "mypage/settings", ...
```

* この2つは、**パスが同じ**で、HTTPメソッドだけが異なる
* `as:` は、**そのルートをパスヘルパーで呼び出すための名前を付けるもの**
* そのため、同じパスに対しては **1回だけ指定すれば足りる**
