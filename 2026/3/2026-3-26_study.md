## Ruby基礎

- [ ] `%w`とは？

👉 ダブルクォートを書かずに文字列の配列を書くときの**Ruby記法**

**解説：**
`%w[...]` は、ダブルクォートを書かずに文字列の配列を作るときに使います。
中の要素は**文字列**として扱われます。式展開はされないので、単純な一覧を書くときに向いています。

```ruby
ALLOWED_TYPES = %w[image/jpeg image/png image/webp]
```

このコードでは、許可するMIMEタイプを文字列の配列として短く書くために `%w` を使っています。
- [ ] `freeze`とは？

👉 **オブジェクトの中身を変更できないようにするメソッド**

**解説：**
`freeze` を呼ぶと、そのオブジェクトは破壊的変更ができなくなります。
定数として使う配列や文字列を、うっかり書き換えないためによく使います。変更しようとするとエラーになることがあります。

```ruby
ALLOWED_TYPES = %w[image/jpeg image/png].freeze
```

このコードでは、許可する型の配列が後から変更されないようにするために `freeze` を使っています。
- [ ] 文字列とシンボルの違いとは？

👉 **文字列は文字そのもの、シンボルは名前やラベルとしてよく使う値**

**解説：**
文字列は `"name"` のように文字データです。
シンボルは `:name` のように、メソッド名・キー名・オプション名など「識別子」としてよく使われます。
見た目は似ていますが、別のものです。

```ruby
"user"   # 文字列
:user    # シンボル
presence:  # シンボルの省略記法

validates :name, presence: true
```

このコードでは、`"user"` は文字データとして、`:name` や `presence:` は名前や設定項目を表すために使っています。
- [ ] `proc`とは？

👉 **あとで実行できる処理を、オブジェクトとして持てるもの**

**解説：**
`proc` は、ブロックをオブジェクトとして扱えるようにしたものです。
変数に入れたり、あとで `call` したりできます。
「条件を後から実行したい」ときによく使います。

```ruby
check = proc { |user| user.admin? }
check.call(current_user)
```

このコードでは、管理者かどうかを判定する処理を、あとで呼べるように `proc` として持たせています。
- 「ブロックを Proc に変換している」イメージ
  - 処理を変数に保存できる
  - 同じ処理を何度でも使える

## Ruby / Railsの条件分岐・書き方の違い

- [ ] `if` と `if:` の違いとは？

👉 `if`** はRubyの条件分岐、**`if:`** はRailsのオプション**

**解説：**
`if` は普通のRubyの文法で、「この条件のときだけ実行する」という意味です。
一方 `if:` は、Railsの `validate` や `validates` などで使う**オプション名**です。
見た目は似ていますが、役割が違います。

```ruby
validate :avatar_size_valid, if: :avatar_attached?

if user.avatar.attached?
  puts "画像あり"
end
```

このコードでは、バリデーションを条件付きで動かすために `if:` を使い
通常の条件分岐をするために `if` を使っています。
- [ ] `if: :メソッド名` と `if: -> { ... }` の違いとは？

👉 **メソッドを呼ぶか、その場で書いた条件式を使うかの違い**

**解説：**
`if: :avatar_attached?` は、モデルのメソッドを呼んで条件判定します。
`if: -> { avatar.attached? }` は、その場でラムダを書いて条件判定します。
再利用しやすさならメソッド、短い条件をその場で書くならラムダが向いています。

```ruby
validate :avatar_size_valid, if: :avatar_attached?
validate :avatar_type_valid, if: -> { avatar.attached? }
```

このコードでは、再利用しやすい条件にはメソッド名を使い、その場だけの短い条件にはラムダを使っています。
-  `ラムダ`とは？

👉 **Procの一種で、より厳密なルールで動く関数オブジェクト**

**解説：**
ラムダは、処理（ブロック）をオブジェクトとして保存できる点は Proc と同じですが、引数の数チェックや return の挙動が厳密です。
Railsでは scope やコールバックなど「あとで実行する処理」を定義するときによく使われます。

```
add = ->(a, b) { a + b }

add.call(2, 3)
# => 5
```

このコードでは、2つの数値を足す処理をあとで実行できるように、ラムダとして保存しています。

## Railsバリデーション

- [ ] `validate` と `validates` の違いとは？

👉 `validate`** は自作の検証メソッド、**`validates`** は用意された検証ヘルパーを書くときに使う**

**解説：**
`validates` は `presence` や `length` など、Railsが用意しているバリデーションを書くときに使います。
`validate` は、自分で作ったメソッドを呼んで独自チェックをしたいときに使います。
標準のチェックか、自作のチェックかで使い分けます。

```ruby
validates :name, presence: true
validate :avatar_size_valid

def avatar_size_valid
  errors.add(:avatar, "が大きすぎます") if avatar.blob.byte_size > 5.megabytes
end
```

このコードでは、標準の必須チェックのために `validates` を使い、独自のサイズチェックのために `validate` を使っています。

## ファイルアップロード・画像・MIME関連

- [ ] `content_type`とは？

👉 **データの種類を表す情報**

**解説：**
`content_type` は、`image/png` や `application/pdf` のように、そのデータが何の種類かを表す値です。
RailsやActive Storageでは、添付ファイルの型チェックでよく使います。
画像かPDFかを見分けたいときに使います。

```ruby
avatar.content_type
# => "image/jpeg"
```

このコードでは、添付された画像がどんな種類のファイルか確認するために `content_type` を使っています。
- [ ] `MIMEタイプ`とは？

👉 **ファイルや通信データの種類を表す決まった名前**

**解説：**
MIMEタイプは、`image/jpeg` や `text/plain` のように「これは何のデータか」を表す識別子です。
HTTPの `Content-Type` にも使われます。
つまり、`content_type` に入っている値の中身が MIMEタイプ です。

```ruby
ALLOWED_TYPES = %w[image/jpeg image/png]
```

このコードでは、アップロードを許可する MIMEタイプ を一覧で管理するために文字列の配列を使っています。
- [ ] 偽装（スプーフィング）とは？

👉 **本当の中身とは、違うふりをさせること**

**解説：**
ファイルアップロードでは、拡張子や `content_type` だけを画像っぽく見せて、実は別の中身を送ることがあります。
そのため、見た目の情報だけで安全だと決めつけるのは危険です。
ファイル検証では、この点に注意が必要です。

```ruby
validates :avatar, content_type: %w[image/png image/jpeg]
```

このコードでは、送られたファイルの種類をチェックするために `content_type` 検証を使っています。
- [ ] `gem 'active_storage_validations'`とは？

👉 **Active Storage の添付ファイルに、使いやすいバリデーションを追加するgem**

**解説：**
これはRails標準そのものではなく、**別途追加するgem**です。
Active Storage の添付ファイルに対して、存在チェックや型チェックなどをわかりやすく書けるようにします。
使うにはGemfileに追加してインストールが必要です。

```ruby
# Gemfile
gem "active_storage_validations"

# app/models/user.rb
validates :avatar, attached: true, content_type: %w[image/png image/jpeg]
```

このコードでは、Active Storage の添付ファイルに対して、存在チェックと型チェックを簡単に書くために `active_storage_validations` を使っています。

## CSS

- [ ] `object-fit: cover`とは？

👉 **画像を枠いっぱいに表示しつつ、はみ出す部分を切り取るCSS**

**解説：**
`object-fit` は、`img` などを枠にどうはめ込むかを決めるCSSです。
`cover` を使うと、縦横比を保ちながら枠を埋めるので、余白が出にくい代わりに一部が切れることがあります。
プロフィール画像の見た目調整でよく使います。

```css
.avatar {
  width: 64px;
  height: 64px;
  object-fit: cover;
}
```

このコードでは、アバター画像を正方形の枠いっぱいに自然に表示するために `object-fit: cover` を使っています。

## View / Helper / 表示まわり

- [ ] `helper`と`decorator`の違いとは？

👉 **helper はビュー用の関数、decorator は表示用ロジックを持つ専用オブジェクト**

**解説：**
helper は、ビューから呼ぶ小さな共通処理をまとめる場所です。Railsの標準機能です。
decorator は、表示用ロジックをオブジェクトとして包む考え方で、Rails標準の特別な機能名ではありません。
Draper などのgemで導入することが多いです。

```ruby
def avatar_image_tag(user)
  image_tag(user.avatar)
end
```

このコードでは、ビューから何度も呼ぶ表示処理をまとめるために helper を使っています。
- [ ] `image_tag`とは？

👉 **画像用の**`<img>`**タグを作るRailsヘルパー**

**解説：**
`image_tag` は、Railsのビューで使うヘルパーです。
画像パスや添付ファイルなどから `<img>` タグを作れます。
`alt` やサイズなどもオプションで指定できます。

```erb
<%= image_tag "icon.png", alt: "アイコン", size: "32x32" %>
```

このコードでは、画像ファイルから `<img>` タグを簡単に作るために `image_tag` を使っています。
- [ ] `style: style`とは？

👉 `style`** というHTML属性に、**`style`** という変数の値を渡している書き方**

**解説：**
これはRailsの特別な機能名ではありません。
Rubyのハッシュで `style:` というキーに、`style` という変数を渡しているだけです。
左側は属性名、右側は変数名です。

```erb
<% style = "width: 32px; height: 32px;" %>
<%= image_tag "icon.png", style: style %>
```

このコードでは、変数に入れたCSS文字列を `<img>` タグの `style` 属性に渡すために `style: style` を使っています。
- [ ] `image_tag`と`tag.img`の違いとは？

👉 `image_tag`** は画像向けの専用ヘルパー、**`tag.img`** は汎用タグ生成の書き方**

**解説：**
`image_tag` は画像用に用意された専用ヘルパーで、普通はこちらを使うことが多いです。
`tag.img` は、汎用のタグ生成ヘルパーで `<img>` を直接組み立てる書き方です。
画像表示なら `image_tag` の方が読みやすいことが多いです。

```erb
<%= image_tag "icon.png", alt: "アイコン" %>
<%= tag.img src: "/images/icon.png", alt: "アイコン" %>
```

このコードでは、画像専用の便利な書き方として `image_tag` を使い、汎用的に `<img>` を組み立てる方法として `tag.img` も使っています。
- [ ] `avatar_image_tag`とは？

👉 **アバター表示用に自分で作ることが多いヘルパーメソッド名**

**解説：**
`avatar_image_tag` 自体は、Railsの特別な機能名ではありません。
多くの場合は、`image_tag` をラップして「アバター画像を表示する共通処理」をまとめるために、アプリ側で自由に定義する名前です。
どんな表示にするかは、アプリ側の実装で決まります。

```ruby
def avatar_image_tag(user, size: 48)
  if user.avatar.attached?
    image_tag user.avatar, size: "#{size}x#{size}", class: "rounded-full object-cover"
  else
    image_tag "default_avatar.png", size: "#{size}x#{size}", class: "rounded-full object-cover"
  end
end
```

このコードでは、アバター画像の表示ルールを1か所にまとめるために `avatar_image_tag` という自作ヘルパーを使っています。
- [ ] `ラップ`とは？

👉 **処理や値を、別のもの（関数・オブジェクトなど）で包むこと**

**解説：**
ラップ（wrap）は、ある処理やデータをそのまま使うのではなく、別の処理で包んで追加の役割（共通処理・制御など）を持たせることです。
RailsやRubyでは、Procやメソッドで処理を包んだり、ヘルパーでHTMLを包む場面でよく使われます。

```
def log_wrapper
  puts "開始"
  result = yield
  puts "終了"
  result
end

log_wrapper do
  puts "処理中"
end
```

このコードでは、処理の前後にログを追加するために、ブロックをラップしています。

## テスト・外部通信

- [ ] `WebMock`とは？

👉 **RubyでHTTP通信を本物の外部通信なしでテストできるライブラリ**

**解説：**
WebMock は、HTTPリクエストをスタブしたり、期待した通信が行われたか確認したりするためのライブラリです。
外部APIや画像URLへの通信を、テスト中だけ偽物に差し替えたいときによく使います。
Rails標準ではなく、別途導入するgemです。

```ruby
stub_request(:get, "https://example.com/avatar.jpg")
  .to_return(status: 200, body: "fake image", headers: { "Content-Type" => "image/jpeg" })
```

このコードでは、外部の画像URLに本当にアクセスせず、テスト用のレスポンスを返すために `WebMock` を使っています。
