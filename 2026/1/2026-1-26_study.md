Lint（リント）

## 目的

- コードの**品質/スタイル**を機械的にチェックして、読みやすさ・保守性を上げる
- 人のレビューを「仕様/設計」に寄せる（スタイル指摘を減らす） ([GitHub](https://github.com/rubocop/rubocop?utm_source=chatgpt.com "rubocop/rubocop: A Ruby static code analyzer and ..."))

## 現象（ログ/画面）

- RuboCopを実行すると、違反（offense）が **Cop名 + メッセージ** で表示される
- `[Correctable]` のように、自動修正できる違反が示されることがある ([RuboCop](https://docs.rubocop.org/rubocop/usage/basic_usage.html?utm_source=chatgpt.com "Basic Usage :: RuboCop Docs"))

## 対応（ステップバイステップ/差分/コマンド）

1. まず実行して現状把握
    

```bash
bundle exec rubocop
```

（オプション・表示などの基本は公式の Basic Usage を参照） ([RuboCop](https://docs.rubocop.org/rubocop/usage/basic_usage.html?utm_source=chatgpt.com "Basic Usage :: RuboCop Docs"))

2. 必要なら自動修正（差分は必ず確認）
    

```bash
bundle exec rubocop -a   # safe autocorrect
bundle exec rubocop -A   # autocorrect all(unsafe含む)
```

([RuboCop](https://docs.rubocop.org/rubocop/usage/basic_usage.html?utm_source=chatgpt.com "Basic Usage :: RuboCop Docs"))

1. プロジェクト方針に合わせて設定（必要な差分だけ書く）
   
- `.rubocop.yml` の `AllCops` や `Exclude` などで対象/除外やルール調整 ([RuboCop](https://docs.rubocop.org/rubocop/configuration.html?utm_source=chatgpt.com "Configuration :: RuboCop Docs"))

## 判断基準（なぜそうした）

- RuboCopは **静的解析（linter）/フォーマッタ** として、規約に沿って問題を検出・一部自動修正できるため、一定品質を継続しやすい ([GitHub](https://github.com/rubocop/rubocop?utm_source=chatgpt.com "rubocop/rubocop: A Ruby static code analyzer and ..."))
    
- 指摘が「Rubyの仕様理解」に絡む場合は、Ruby公式RDocで挙動確認する（例：`String#upcase!` が変更なしだと `nil` を返す等） ([Rubyリファレンスガイド](https://docs.ruby-lang.org/en/3.1/String.html?utm_source=chatgpt.com "class String - RDoc Documentation"))
