# 今日理解できたこと
## ① ドット記法とブラケット記法の違い

* ドット記法
  → キーが固定のときに使う
  → シンプルで読みやすい

* ブラケット記法
  → 変数を使って動的にキーを指定できる
  → キーが後から決まるときに使う

### 本質

「アクセスするキーが最初から決まっているか、それとも後から決まるか」で使い分ける。

---

## ② Web以前とWebの違い

* Web以前
  → 高機能
  → 複雑
  → 閉じた世界で使われることが多かった

* Web
  → シンプル
  → 単方向リンク
  → インターネット全体を前提にしている

### 本質

複雑で高機能な仕組みよりも、シンプルで扱いやすい仕組みのほうが広まりやすい。

---

## ③ 分散システムの問題

* CORBA / DCOMは強力な仕組みだった
* ただし複雑で、異なる環境では広く使いにくかった
* 世界規模でつながる仕組みとしては扱いづらかった

### それに対してWebは

HTTPというシンプルな共通ルールを使うことで、異なるOSや環境でも利用しやすくなった。

---

## ④ Stimulusの仕組み

### data-controller

HTMLとJavaScriptをつなぐ入口。
「このHTML要素を、どのコントローラーが担当するか」を決める。

例：

```html
<div data-controller="hello">
```

---

### data-action

イベントとメソッドを結びつける仕組み。
「何をしたら、何を実行するか」を決める。

例：

```html
<button data-action="click->hello#greet">Greet</button>
```

---

### targets

JavaScriptからHTML要素を簡単に取得する仕組み。

HTML側：

```html
<input data-hello-target="name" type="text">
```

Controller側：

```javascript
static targets = ["name"]
```

これによって、

```javascript
this.nameTarget
```

と書くだけで、そのHTML要素を取得できる。

### 本質

`this.nameTarget` は「特定のHTML要素へのショートカット」である。
