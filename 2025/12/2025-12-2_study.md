## 学習内容

### SQL

##### 自動コミットモードとは？

説明：
**1 つの SQL 文を実行するたびに自動的に `COMMIT` が実行されるモード**のことです。

■ トランザクションには 2 つの動作モードがあります！

|モード|動作|
|---|---|
|**自動コミットモード（auto-commit mode）**|各 SQL が終わるたびに自動で `COMMIT` される|
|**手動コミットモード（手動トランザクション）**|`BEGIN` → SQL 実行 → `COMMIT/ROLLBACK` を自分で操作する|

具体例１：自動コミットモード
```sql
UPDATE users SET name = 'Alice' WHERE id = 1; -- この UPDATE が終わった瞬間に自動で COMMIT が実行される
```
そのため、**失敗したときにまとめて巻き戻す（ROLLBACK）ということができない**。

具体例２；手動トランザクション
```sql
BEGIN;

UPDATE users SET name = 'Alice' WHERE id = 1;
UPDATE users SET age = 20 WHERE id = 1;

COMMIT; -- ← 自分で確定する

```
途中で失敗したら、`ROLLBACK;`で全て元に戻せる。

##### 補足
- 自動コミットモードを解除するには、ツールや環境よって違います💦
	- MySQLだと、`SET AUTOCOMMIT=0`
