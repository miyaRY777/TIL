## 学習内容

### SQL

##### 左外部結合

説明：
左表と右表があるとします！
左表において、結合相手が見つからなかったり、NULLであっても
必ず出力せよ！っと指示を出すのが左外部結合です！

基本形：
```sql
SELECT ~ 
FROM 左表の名前
LEFT JOIN 右表の名前
ON 結合条件

```

動き：
```sql
/*家計簿テーブル（左表）*/
|日付|費目ID|メモ|
|2024-02-03|2|カフェラテ|
|2024-02-03|4|昼食|
|2024-02-05|NULL|昼食|

/*費目テーブル（右表）*/
|ID|名前|
|1|給料|
|2|食費|
|NULL|その他|
```

1. 左表より、結合相手の行がなかった場合でも、処理は諦めません！
2. この場合、右表から結合されていない行の内容を全てNULLにして結合します！
```sql
/*処理結果*/
|日付|費目ID|メモ|ID|名前|
|2024-02-03|2|カフェラテ|2|食費|
|2024-02-03|4|昼食|NULL|NULL|
|2024-02-05|NULL|昼食|NULL|NULL|
```

補足
左外部結合は
NULLの行を生み出してでも、左表のすべての行を必ず出力する処理のことでした！

これの右表verが右外部結合
```sql
SELECT ~ 
FROM 左表の名前
RIGHT JOIN 右表の名前
ON 結合条件
```

左右の表のすべての行を必ず出力する完全外部結合
```sql
SELECT ~ 
FROM 左表の名前
FULL JOIN 右表の名前
ON 結合条件
```


### まとめ

- 外部結合：左外部結合、右外部結合、完全外部結合の総称
- 内部結合：結合すべき相手の行が見つからない場合に行が消滅してしまう通常の結合
