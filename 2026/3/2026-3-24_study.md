# 今日やったこと

Active Job の基本（enqueue / perform_later / perform_now）と、
ジョブの引数がどのように扱われるか（Global ID・シリアライズ・デシリアライズ）について整理した。

---

# 今日理解できたこと

Active Job は「あとで実行する処理」を扱う仕組みで、`perform_later` を使うことでジョブをキューに登録できると理解できた。
一方で `perform_now` はキューを使わず、その場で処理を実行する同期処理であることも整理できた。

また、モデル（Userなど）をそのままジョブに渡せる理由は Global ID にあり、
実際にはオブジェクトそのものではなく「復元できる情報」に変換されて保存される仕組みだと分かった。

さらに、その過程で

* シリアライズ：保存できる形に変換
* デシリアライズ：元のオブジェクトに戻す
  という流れが内部で自動的に行われていることを理解できた。

# 感想など

「ジョブ＝裏で動く処理」というイメージがかなり明確になった。
特に Global ID によって「オブジェクトそのまま渡してるように見えるけど、実際は違う」という仕組みが理解できたのが大きい。

今まではなんとなく `perform_later` を使っていたが、
「enqueue → シリアライズ → 保存 → デシリアライズ → 実行」という流れで考えられるようになったので、理解が一段深まったと感じる。
